- If BPB catches up to baseline → throughput optimization is the priority
- If BPB stays behind AND throughput is 1.7x slower → AttnRes is a net loss at d12

### If AttnRes needs throughput optimization (recommended fixes, priority order):
1. **Cache completed blocks as stacked tensor** — stop rebuilding Python lists and restacking every sub-layer. Pre-compute normed keys at block boundaries.
2. **Batch inter-block queries** — within each block, all 6 sub-layer queries share the same completed block reps. Precompute once per block instead of 6 times.
3. **Reduce `attn_res_num_blocks`** from 4 to 2 as a quick knob to halve overhead.
4. **Two-phase block schedule** (paper's Section 4.2) — one batched inter-block pass per block, then cheap sequential merges via online softmax.

### If AttnRes is a net loss at d12:
- Pivot to DCA (arXiv:2502.06785) — complementary mechanism, 3x convergence claim, pure PyTorch, no kernel changes
- Or try AttnRes at d24 where depth signal dilution is more severe and the 1.25x compute advantage may overcome the throughput overhead

## Context & Notes

### Key Findings from x0 Re-Injection Experiment
- Flat 1.0 init produced BPB 0.8548 vs baseline 0.8538 — no improvement
- Learned curve is non-monotonic: L0=1.67, L8=2.10 (spike), L9=-0.54, L11=-0.91 (negative — model subtracts input)
- resid_lambdas all dropped from 1.15→1.05 init to 0.42–0.95 range
- **Conclusion:** x0 re-injection is a soft knob, not a bottleneck. The optimizer compensates regardless of init.

### AttnRes Throughput Diagnosis
- FA3 confirmed active — NOT an SDPA fallback issue
- Root cause: `_attn_res()` called 25 times per forward, each doing `torch.stack` + `norm` on the full source list
- `torch.compile` captures the whole forward in one graph with zero graph breaks
- The bottleneck is pure memory traffic: stacking and norming ~25 GB of data per forward pass

- The code review noted a math.ceil→floor division fix was applied (matches paper's block sizing)
- AttnRes queries use `query.to(K.dtype)` for torch.compile safety with mixed fp32/bf16
