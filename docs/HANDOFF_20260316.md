## Key Findings from This Session

### 1. 24K tokenizer hurts CORE (reverted)
At d24 scale (D=1536), reducing vocab from 32K to 24K improved D/V from 0.047 to 0.063 — marginal gradient flow gain. But fewer bytes per token means less content per 2048-token sequence, hurting knowledge-heavy CORE tasks (MMLU, ARC). CORE 0.2542 fell just short of 0.2565 threshold. Reverted to 32K.

### 2. NCA was using wrong config
We used n=4 (256 tokens) and LR 3e-4 based on a secondary paper finding about scaling to 1.6B params. The paper's headline results (1.6x speedup) used n=10 (10,000 tokens) with LR 1e-4. Switched to paper defaults. Sweep in progress to find optimal step count.

### 3. D-24 OOM constraints
D-24 + FP8 uses ~77.7GB/GPU. device-batch-size=32 OOMs. NCA batch-size=32 also OOMs. Fixed at device-batch-size=16 (grad accum 2) and nca-batch-size=8.

### 4. Head avoidance shows promise
The gradient bottleneck paper (Godey & Artzi 2026, arXiv:2603.10145) shows 95-99% of gradient norm is destroyed by the LM head. Our proxy loss bypass (cosine similarity against lm_head.weight[targets]) reached BPB 0.887 on d12 — close to NCA's 0.863. Needs matched baseline comparison (running now on H200).

### 5. torchrun --run flag conflict
`--run=` is ambiguous with torchrun's `--run-path`. Drop the flag or use `python -m torch.distributed.run`.

## Context & Notes

### H100/H200 Instance Setup
- **System deps:** `apt-get install python3-dev` (Triton needs it)
- **Disk workaround (8xH100):** Container has 16GB overlay. Symlink `~/.cache/nanochat → /dev/shm/nanochat_cache`
- **Torch compile cache:** `/tmp/torchinductor_root` consumes ~7GB, may fill overlay. Periodically clear or route to /dev/shm
- **TORCH_DISABLE_ADDR2LINE=1:** Suppresses noisy C++ stack trace warnings
- **uv run** prefix for all Python commands
- **BPB monitoring:** `tail -f ~/.cache/nanochat/bpb_log.csv` (new feature, needs latest master)

### Head Avoidance Implementation Notes (branch: exp/head-avoidance)
- Proxy path uses `orig_model` (uncompiled) to avoid torch.compile recompilation
- Proxy target is `lm_head.weight[y]` (decoder space), NOT `wte(y)` (input embeddings) — model uses untied weights
- DDP-safe: zero-fills None grads before optimizer step
- FP8-safe: dequantizes lm_head.weight to compute dtype before indexing
- EMA loss only updates on CE steps; `ce_step_count` tracked separately for debias
- `ce_step_count` is saved to checkpoint for correct resume behavior
- Codex review identified and fixed: DDP grad sync issue, wrong target space (wte→lm_head), FP8 precision

### Important Gotchas
- `--core-metric-every=-1` disables CORE entirely, INCLUDING the final step. Use a large number (10000) instead to get CORE only at the end
- `--sample-every=-1` disables text generation sampling (fine for benchmarking)
- NCA batch size must be 8 at d24 (32 OOMs)
- modifications.md exists in TWO locations — keep both in sync (see sync note at top of file)
- The `knowledge/` directory in the Speedrun repo is untracked (in .gitignore or just not added). Paper summaries live there.
